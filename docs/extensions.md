# Writing Widget Extensions

Guide for building custom widget extensions for plushie. Extensions
let you render arbitrary Rust-native widgets while keeping your app's
state and logic in TypeScript.

## Quick start

An extension has two halves:

1. **TypeScript side:** use `defineExtensionWidget()` to declare the
   widget's props, commands, and (for native widgets) the Rust crate.

2. **Rust side:** implement the `WidgetExtension` trait from
   `plushie-core`. This receives tree nodes from TypeScript and
   returns `iced::Element`s for rendering.

```typescript
// lib/sparkline.ts
import { defineExtensionWidget, extensionCommands } from 'plushie'

export const sparklineConfig = {
  type: 'sparkline',
  props: {
    data: { list: 'number' } as const,
    color: 'color' as const,
    capacity: 'number' as const,
  },
  events: ['sample_clicked'],
  commands: ['push'],
  rustCrate: 'native/my_sparkline',
  rustConstructor: 'my_sparkline::SparklineExtension::new()',
} as const

export const Sparkline = defineExtensionWidget(sparklineConfig)
export const SparklineCmds = extensionCommands(sparklineConfig)
```

```rust
// native/my_sparkline/src/lib.rs
use plushie_core::prelude::*;

pub struct SparklineExtension;

impl SparklineExtension {
    pub fn new() -> Self { Self }
}

impl WidgetExtension for SparklineExtension {
    fn type_names(&self) -> &[&str] { &["sparkline"] }
    fn config_key(&self) -> &str { "sparkline" }

    fn render<'a>(
        &self, node: &'a TreeNode, env: &WidgetEnv<'a>,
    ) -> Element<'a, Message> {
        let data = prop_f32_array(node.props(), "data").unwrap_or_default();
        // ... render using iced widgets ...
        text("sparkline placeholder").into()
    }
}
```

Build with `npx plushie build`. Extensions are registered in
`plushie.extensions.json`:

```json
{
  "extensions": [
    {
      "type": "sparkline",
      "rustCrate": "native/my_sparkline",
      "rustConstructor": "my_sparkline::SparklineExtension::new()",
      "props": { "data": { "list": "number" }, "color": "color" },
      "events": ["sample_clicked"],
      "commands": ["push"]
    }
  ]
}
```

The build generates a Cargo workspace with a `main.rs` that chains
`.extension()` calls:

```rust
// Auto-generated by npx plushie build
plushie::run(
    PlushieAppBuilder::new()
        .extension(my_sparkline::SparklineExtension::new())
)
```

## Extension kinds

### Native widget extensions (Rust-backed)

For widgets rendered by a Rust crate. Requires `rustCrate` and
`rustConstructor` in the config.

```typescript
const HexView = defineExtensionWidget({
  type: 'hex_view',
  props: { data: 'string', columns: 'number' },
  rustCrate: 'native/hex_view',
  rustConstructor: 'hex_view::HexViewExtension::new()',
})
```

### Pure TypeScript composite widgets

For widgets composed entirely from existing plushie widgets. No Rust
code needed. Just define a function that returns a UINode tree:

```typescript
import { canvas } from 'plushie/ui'
import { rect, circle, linearGradient } from 'plushie/canvas'

function donutChart(
  id: string,
  segments: Array<{ label: string; value: number; color: string }>,
  opts?: { size?: number; thickness?: number },
): UINode {
  const size = opts?.size ?? 200
  // Build canvas shapes from segments...
  return canvas(id, { width: size, height: size }, [
    // arc shapes via path commands...
  ])
}
```

Pure TypeScript widgets don't need `defineExtensionWidget` at all --
they're just functions that return UINodes. Use
`defineExtensionWidget` only when you need handler registration,
command generation, or Rust build integration.

## Using extensions in your app

Extension widgets produce the same UINode structure as built-in
widgets. Use them in `view()` like any other widget:

```typescript
view: (state) => (
  <Window id="main" title="Dashboard">
    <Column spacing={16}>
      {Sparkline('spark-1', {
        data: state.samples,
        color: '#4CAF50',
        capacity: 100,
        onSampleClicked: handleSampleClick,
      })}
      <Button id="push" onClick={(s) => [
        s,
        SparklineCmds.push('spark-1', { value: Math.random() * 100 }),
      ]}>
        Add Sample
      </Button>
    </Column>
  </Window>
),
```

The runtime normalizes extension nodes identically to built-in ones.
The renderer uses the `type` string to dispatch to the registered
extension.

## Extension config

Pass configuration to extensions at startup:

```typescript
app({
  settings: {
    extensionConfig: {
      sparkline: { defaultColor: '#4CAF50', maxSamples: 1000 },
    },
  },
  // ...
})
```

The renderer forwards `extension_config` to each registered extension
via the `init()` trait method. The config key must match the Rust
extension's `config_key()` return value.

## Extension commands

Commands bypass the tree diff/patch cycle and go directly to the
Rust extension's `handle_command()` method. Use them for
high-frequency data (streaming values, log lines, terminal output).

```typescript
// Generated command constructors:
const cmds = extensionCommands(sparklineConfig)
cmds.push('spark-1', { value: 42 })

// Or use the raw Command API:
Command.extensionCommand('spark-1', 'push', { value: 42 })

// Batch multiple commands (processed in one cycle):
Command.extensionCommands([
  { nodeId: 'spark-1', op: 'push', payload: { value: 42 } },
  { nodeId: 'spark-2', op: 'clear', payload: {} },
])
```

## Extension events

Extension events arrive as `WidgetEvent` with a custom `type` string
matching the Rust extension's event family:

```typescript
update(state, event) {
  if (isWidget(event) && event.type === 'sample_clicked'
      && event.id === 'spark-1') {
    const data = event.data as Record<string, unknown>
    return { ...state, selectedSample: data['index'] as number }
  }
  return state
}
```

Or with inline handlers via `defineExtensionWidget`:

```typescript
Sparkline('spark-1', {
  data: state.samples,
  onSampleClicked: (s, e) => ({
    ...s,
    selectedSample: (e.data as Record<string, unknown>)['index'] as number,
  }),
})
```

## Prop types

| Type | TypeScript | Wire format |
|---|---|---|
| `'string'` | string | string |
| `'number'` | number | number |
| `'boolean'` | boolean | boolean |
| `'color'` | Color | hex string |
| `'length'` | Length | number/string/object |
| `'padding'` | Padding | number/array |
| `'font'` | Font | string/object |
| `'alignment'` | Alignment | string |
| `'style'` | StyleMap | string/object |
| `'any'` | unknown | passed through |
| `{ list: type }` | array | array |

The `a11y` and `eventRate` props are available on all extension
widgets automatically.

## Container extensions

Set `container: true` to accept children:

```typescript
const Panel = defineExtensionWidget({
  type: 'panel',
  props: { title: 'string', collapsible: 'boolean' },
  container: true,
})

// Usage (children as third argument):
Panel('settings', { title: 'Settings', collapsible: true }, [
  text('Option 1'),
  text('Option 2'),
])
```

## The Rust side: WidgetExtension trait

The trait has three required methods and several optional ones.
Choose the tier that fits.

### Tier A: render-only

Implement `type_names`, `config_key`, and `render`. Good for
widgets that compose existing iced widgets with no Rust-side state.

```rust
impl WidgetExtension for HexViewExtension {
    fn type_names(&self) -> &[&str] { &["hex_view"] }
    fn config_key(&self) -> &str { "hex_view" }

    fn render<'a>(
        &self, node: &'a TreeNode, env: &WidgetEnv<'a>,
    ) -> Element<'a, Message> {
        let data = node.prop_str("data").unwrap_or_default();
        container(text(data)).into()
    }
}
```

### Tier B: interactive (+handle_event)

Add `handle_event` to intercept events before they reach TypeScript.
Returns `EventResult` controlling event flow:

```rust
fn handle_event(
    &mut self,
    node_id: &str,
    family: &str,
    data: &Value,
    caches: &mut ExtensionCaches,
) -> EventResult {
    match family {
        "canvas_press" => {
            let x = data.get("x").and_then(|v| v.as_f64()).unwrap_or(0.0);
            // Transform raw canvas event into a semantic one
            EventResult::Consumed(vec![
                OutgoingEvent::extension_event(
                    "plot_click".to_string(),
                    node_id.to_string(),
                    Some(json!({"plot_x": x})),
                )
            ])
        }
        "canvas_move" => {
            // Handle internally, don't forward
            EventResult::Consumed(vec![])
        }
        _ => EventResult::PassThrough,
    }
}
```

### EventResult variants

| Variant | Behavior |
|---|---|
| `PassThrough` | Forward the original event to TypeScript as-is (default) |
| `Consumed(events)` | Swallow the original; optionally emit different events |
| `Observed(events)` | Forward the original AND emit additional events |

### Tier C: full lifecycle

Add `prepare`, `handle_command`, and `cleanup` for mutable state:

```rust
fn prepare(
    &mut self, node: &TreeNode,
    caches: &mut ExtensionCaches, theme: &Theme,
) {
    let props = node.props();
    let state = caches.get_or_insert::<SparklineState>(
        self.config_key(), &node.id,
        || SparklineState::new(prop_usize(props, "capacity").unwrap_or(100))
    );
    state.color = prop_color(props, "color");
}

fn handle_command(
    &mut self, node_id: &str, op: &str,
    payload: &Value, caches: &mut ExtensionCaches,
) -> Vec<OutgoingEvent> {
    match op {
        "push" => {
            if let Some(state) = caches.get_mut::<SparklineState>(
                self.config_key(), node_id,
            ) {
                if let Some(v) = payload.get("value").and_then(|v| v.as_f64()) {
                    state.push(v as f32);
                    state.generation.bump();
                }
            }
            vec![]
        }
        _ => vec![],
    }
}

fn cleanup(&mut self, node_id: &str, caches: &mut ExtensionCaches) {
    caches.remove(self.config_key(), node_id);
}
```

### Trait method reference

| Method | Required | Description |
|---|---|---|
| `type_names` | yes | Widget type names this extension handles |
| `config_key` | yes | Key for extension_config lookup |
| `init` | no | Called once at startup with config |
| `prepare` | no | Called before each render pass (mutable state sync) |
| `render` | yes | Returns iced Element for the widget |
| `handle_event` | no | Intercepts events before they reach TypeScript |
| `handle_command` | no | Handles extension_command messages |
| `cleanup` | no | Called when a node is removed from the tree |

## Prop helpers (Rust)

Read props from `TreeNode`. Two equivalent styles:

```rust
// Method style (on TreeNode directly):
let data = node.prop_str("data").unwrap_or_default();

// Free function style (takes node.props()):
let data = prop_str(node.props(), "data").unwrap_or_default();
```

Available helpers from `plushie_core::prelude`:

| Method on TreeNode | Free function | Returns |
|---|---|---|
| `node.prop_str(key)` | `prop_str(node.props(), key)` | `Option<String>` |
| `node.prop_f32(key)` | `prop_f32(node.props(), key)` | `Option<f32>` |
| -- | `prop_f64(node.props(), key)` | `Option<f64>` |
| `node.prop_bool(key)` | `prop_bool(node.props(), key)` | `Option<bool>` |
| `node.prop_color(key)` | `prop_color(node.props(), key)` | `Option<iced::Color>` |
| -- | `prop_length(node.props(), key, fallback)` | `Length` |
| -- | `prop_f32_array(node.props(), key)` | `Option<Vec<f32>>` |
| -- | `prop_range_f32(node.props())` | `RangeInclusive<f32>` |
| `node.prop_padding()` | `prop_padding(node.props())` | `Option<Padding>` |

## Event rate limiting

Extension events support the same rate limiting as built-in widgets.
Mark events with `CoalesceHint`:

```rust
// Latest value wins (position tracking):
let event = OutgoingEvent::extension_event("cursor_pos", node_id, data)
    .with_coalesce(CoalesceHint::Replace);

// Deltas sum (scroll, velocity):
let event = OutgoingEvent::extension_event("pan_scroll", node_id, data)
    .with_coalesce(CoalesceHint::Accumulate(
        vec!["delta_x".into(), "delta_y".into()]
    ));
```

Control frequency from TypeScript via `eventRate`:

```typescript
Sparkline('spark-1', { data: state.samples, eventRate: 30 })
```

## Building with extensions

### Config file

Create `plushie.extensions.json` at your project root:

```json
{
  "extensions": [
    {
      "type": "sparkline",
      "rustCrate": "native/my_sparkline",
      "rustConstructor": "my_sparkline::SparklineExtension::new()",
      "props": { "data": { "list": "number" }, "color": "color" },
      "commands": ["push"]
    }
  ],
  "binaryName": "my-app-plushie"
}
```

### Build command

```sh
PLUSHIE_SOURCE_PATH=~/plushie npx plushie build
PLUSHIE_SOURCE_PATH=~/plushie npx plushie build --release
```

The build:
1. Reads `plushie.extensions.json`
2. Validates extensions (type collisions, crate collisions)
3. Checks Rust toolchain version (>= 1.92.0)
4. Generates a Cargo workspace with `Cargo.toml` and `main.rs`
5. Runs `cargo build`
6. Installs the binary to `node_modules/.plushie/bin/`

### Validation

The build validates:
- Every native extension has both `rustCrate` and `rustConstructor`
- No two extensions use the same type name
- No two extensions use the same crate directory name
- Constructor expressions match valid Rust identifier patterns

## Pure TypeScript widget packages

For widgets composed from built-in primitives. No Rust code, no
rebuild needed.

### When pure TypeScript is enough

Canvas shape builders cover custom 2D rendering. Overlay enables
dropdowns and popovers. StyleMap provides visual customization.
Layout composition covers cards, tab bars, sidebars, and toolbars.
See [Composition patterns](composition-patterns.md) for examples.

### Package structure

```
my-widget/
  src/
    donut-chart.ts
    index.ts
  test/
    donut-chart.test.ts
  package.json
```

The package lists `plushie` as a peer dependency. Consumers don't
need to rebuild anything.

## Panic isolation

The renderer wraps all mutable extension calls in `catch_unwind`:
1. Panic is logged
2. Extension marked as "poisoned"
3. Subsequent calls skipped; render returns error placeholder
4. Poisoned state clears on next Snapshot

A bug in one extension cannot crash the renderer or affect others.

## Canvas cache and GenerationCounter

`iced::widget::canvas::Cache` is `!Send + !Sync` and cannot be
stored in `ExtensionCaches`. Use `GenerationCounter` instead:

```rust
// In ExtensionCaches (Send + Sync):
struct SparklineData {
    samples: Vec<f32>,
    generation: GenerationCounter,
}

// In canvas Program::State (not Send, managed by iced):
struct SparklineState {
    last_generation: u64,
    cache: canvas::Cache,
}
```

Bump the generation when data changes. In `draw()`, compare
generations and clear the cache when stale.

## ExtensionCaches reference

| Method | Description |
|---|---|
| `get::<T>(ns, key)` | Immutable access |
| `get_mut::<T>(ns, key)` | Mutable access |
| `get_or_insert::<T>(ns, key, default_fn)` | Initialize if absent |
| `insert::<T>(ns, key, value)` | Overwrites existing |
| `remove(ns, key)` | Returns whether key existed |
| `contains(ns, key)` | Check existence |

Common keys: `(config_key, node_id)` for per-node state,
`(config_key, "_global")` for global state.

## Complete worked example: Gauge widget

A full native extension showing both TypeScript and Rust sides.

### TypeScript side

```typescript
// src/gauge.ts
import { defineExtensionWidget, extensionCommands } from 'plushie'
import type { ExtensionWidgetConfig } from 'plushie'

export const gaugeConfig: ExtensionWidgetConfig = {
  type: 'gauge',
  props: {
    value: 'number',
    min: 'number',
    max: 'number',
    color: 'color',
    label: 'string',
    width: 'length',
    height: 'length',
  },
  events: ['value_changed'],
  commands: ['set_value', 'animate_to'],
  rustCrate: 'native/gauge',
  rustConstructor: 'gauge::GaugeExtension::new()',
}

/** Gauge widget builder. */
export const Gauge = defineExtensionWidget(gaugeConfig)

/** Gauge command constructors. */
export const GaugeCmds = extensionCommands(gaugeConfig)
```

```typescript
// src/app.tsx -- using the gauge in an app
import { app, Command, isWidget } from 'plushie'
import { Window, Column, Row, Text, Button, Slider } from 'plushie/ui'
import { Gauge, GaugeCmds } from './gauge.js'

interface Model {
  temperature: number
  targetTemp: number
}

export default app<Model>({
  init: { temperature: 20, targetTemp: 20 },

  settings: {
    extensionConfig: {
      gauge: { arcWidth: 8, tickCount: 10 },
    },
  },

  update(state, event) {
    // Handle events from the gauge extension
    if (isWidget(event) && event.type === 'value_changed'
        && event.id === 'temp') {
      const data = event.data as Record<string, unknown>
      return { ...state, temperature: data['value'] as number }
    }
    return state
  },

  view: (state) => (
    <Window id="main" title="Temperature">
      <Column padding={24} spacing={16} alignX="center">
        {Gauge('temp', {
          value: state.temperature,
          min: 0,
          max: 100,
          color: state.temperature > 80 ? '#e74c3c' : '#3498db',
          label: `${Math.round(state.temperature)}°C`,
          width: 200,
          height: 200,
          eventRate: 30,
        })}

        <Slider id="target" value={state.targetTemp} range={[0, 100]}
          onSlide={(s, e) => [
            { ...s, targetTemp: e.value as number },
            GaugeCmds.animate_to('temp', { value: e.value }),
          ]} />

        <Row spacing={8}>
          <Button id="reset" onClick={(s) => [
            { ...s, targetTemp: 20 },
            GaugeCmds.set_value('temp', { value: 20 }),
          ]}>Reset</Button>
        </Row>
      </Column>
    </Window>
  ),
})
```

### Rust side

```rust
// native/gauge/src/lib.rs
use plushie_core::prelude::*;
use serde_json::json;

pub struct GaugeExtension;

impl GaugeExtension {
    pub fn new() -> Self { Self }
}

/// Per-node state stored in ExtensionCaches.
struct GaugeState {
    current_value: f32,
    target_value: f32,
    generation: GenerationCounter,
}

impl GaugeState {
    fn new(value: f32) -> Self {
        Self {
            current_value: value,
            target_value: value,
            generation: GenerationCounter::new(),
        }
    }
}

impl WidgetExtension for GaugeExtension {
    fn type_names(&self) -> &[&str] { &["gauge"] }
    fn config_key(&self) -> &str { "gauge" }

    fn init(&mut self, ctx: &InitCtx<'_>) {
        // Read extension_config if needed
        if let Some(config) = ctx.config {
            // config contains { "arcWidth": 8, "tickCount": 10 }
        }
    }

    fn prepare(
        &mut self, node: &TreeNode,
        caches: &mut ExtensionCaches, _theme: &Theme,
    ) {
        let props = node.props();
        let value = prop_f32(props, "value").unwrap_or(0.0);
        let state = caches.get_or_insert::<GaugeState>(
            self.config_key(), &node.id,
            || GaugeState::new(value),
        );
        // Sync from TypeScript props
        state.current_value = value;
    }

    fn render<'a>(
        &self, node: &'a TreeNode, _env: &WidgetEnv<'a>,
    ) -> Element<'a, Message> {
        let props = node.props();
        let value = prop_f32(props, "value").unwrap_or(0.0);
        let min = prop_f32(props, "min").unwrap_or(0.0);
        let max = prop_f32(props, "max").unwrap_or(100.0);
        let color = prop_color(props, "color")
            .unwrap_or(iced::Color::from_rgb(0.2, 0.5, 0.8));
        let label = prop_str(props, "label").unwrap_or_default();
        let w = prop_length(props, "width", Length::Fixed(200.0));
        let h = prop_length(props, "height", Length::Fixed(200.0));

        // Build the gauge using iced widgets
        let pct = ((value - min) / (max - min)).clamp(0.0, 1.0);
        let display = format!("{:.0}%", pct * 100.0);

        container(
            column![
                text(&label).size(16),
                text(&display).size(32).color(color),
            ]
            .align_x(iced::alignment::Horizontal::Center)
        )
        .width(w)
        .height(h)
        .center(iced::Length::Fill)
        .into()
    }

    fn handle_command(
        &mut self, node_id: &str, op: &str,
        payload: &Value, caches: &mut ExtensionCaches,
    ) -> Vec<OutgoingEvent> {
        match op {
            "set_value" => {
                if let Some(state) = caches.get_mut::<GaugeState>(
                    self.config_key(), node_id,
                ) {
                    if let Some(v) = payload.get("value").and_then(|v| v.as_f64()) {
                        state.current_value = v as f32;
                        state.generation.bump();

                        // Notify TypeScript of the change
                        return vec![OutgoingEvent::extension_event(
                            "value_changed".to_string(),
                            node_id.to_string(),
                            Some(json!({"value": v})),
                        )];
                    }
                }
                vec![]
            }
            "animate_to" => {
                if let Some(state) = caches.get_mut::<GaugeState>(
                    self.config_key(), node_id,
                ) {
                    if let Some(v) = payload.get("value").and_then(|v| v.as_f64()) {
                        state.target_value = v as f32;
                        state.generation.bump();
                    }
                }
                vec![]
            }
            _ => vec![],
        }
    }

    fn cleanup(&mut self, node_id: &str, caches: &mut ExtensionCaches) {
        caches.remove(self.config_key(), node_id);
    }
}
```
